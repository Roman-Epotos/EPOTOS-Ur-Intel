2. ТЕХНОЛОГИЧЕСКИЙ СТЕК
=============================================================
Next.js 14, TypeScript, Tailwind CSS, Supabase (cloud) + pgvector
AI основная: OpenRouter (google/gemini-2.5-flash)
AI PDF сравнение: OpenRouter (anthropic/claude-sonnet-4-5) — ПЛАТНАЯ! только ai-document-compare
Редактор: OnlyOffice (https://office.epotos-port.ru)
Storage: Supabase (buckets: contracts PUBLIC, templates PUBLIC, counterparty-docs PUBLIC)
Хостинг: Vercel Pro (оплачен и подключен с 03.07.2026)
Генерация .docx: JSZip (прямая замена {{field}} в XML)
PDF извлечение текста: unpdf — нестабилен на Vercel! Показываем ошибку
Связанный модуль: ЭПОТОС-Core (https://epotos-core.vercel.app) — отдельный
  деплой, хостит RAG-ассистента (/api/assistant/chat, /api/assistant/search)
  на pgvector, используется ЮрИнтел через простой fetch (без общего кода)

⚠️ КРИТИЧЕСКИ ВАЖНО (обнаружено 07.07.2026): OpenRouter с 27.06.2026
  заблокировал API-запросы от аккаунтов из российского региона (не только
  платежи, как было раньше с 11.05.2026, — теперь блокируются САМИ ЗАПРОСЫ).
  ЮрИнтел ПОКА НЕ ЗАТРОНУТ, так как запросы к OpenRouter физически идут с
  серверов Vercel (нероссийский IP) — подтверждено проверкой, всё работает
  стабильно. НО: если в будущем переносить бэкенд/серверную часть на
  российский хостинг (см. раздел 12, Selectel) — это СЛОМАЕТ весь
  ИИ-функционал, если не предусмотреть отдельно нероссийский маршрут для
  вызовов к OpenRouter. Блокировка похожа на IP-based (не account-based) —
  раз через Vercel всё ещё работает.

  максимуме, его номер не переиспользуется; при безвозвратном удалении
  номер того года остаётся неиспользованным пробелом — это ожидаемо.

  между select максимума и insert возможна гонка (race condition) при
  одновременном создании двух документов — см. раздел 12, пункт 6
ВАЖНО: Э-К — двойной префикс → company_prefix из БД или startsWith('Э-К')

Статусы: черновик → на_согласовании → согласован → [на_подписи_в_эдо →]
  загружен_частично → подписан → на_исполнении (↘ отклонён)
LOCKED_STATUSES = ['согласован','на_подписи_в_эдо','загружен_частично','подписан','на_исполнении']

Приём на хранение (storage_confirmed_*) НЕ является статусом — это
  независимый бейдж поверх статуса, доступен когда status IN
  ('подписан','на_исполнении')
